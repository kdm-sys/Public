## 1. 이 매뉴얼에 대해

### 왜 VS Code + Claude Code인가?

일반 AI 채팅(ChatGPT, Claude 웹)은 대화창에 텍스트를 주고받는 방식입니다.  
**VS Code + Claude Code** 조합은 다릅니다:

| 일반 AI 채팅 | VS Code + Claude Code |
|---|---|
| 한 번에 하나의 대화 | 프로젝트 폴더 전체를 맥락으로 활용 |
| 이전 대화를 까먹음 | 파일에 저장된 설정을 항상 참조 |
| 복사-붙여넣기 반복 | 파일을 직접 생성/수정 |
| 문서 간 일관성 유지 어려움 | 여러 문서를 동시에 참조하며 일관성 유지 |

**즉, AI가 야구매니저의 선수 스탯, 시뮬레이션 공식, 콘텐츠 구조를 전부 이해한 상태에서 기획서를 작성합니다.**

### 이 매뉴얼에서 다루는 것

- VS Code와 Claude Code 설치 방법
- 게임 기획 프로젝트 폴더 세팅
- AI가 참조할 규칙 파일(CLAUDE.md) 작성법
- 실제 기획서 작성 예시 (야구매니저 기준)

### 사전 지식

- **프로그래밍 지식: 불필요** ❌
- 컴퓨터에서 폴더를 만들고 파일을 저장할 수 있으면 충분합니다
- 대부분의 작업은 **마우스 클릭과 한국어 입력**만으로 진행됩니다

## 2. 설치하기

총 2가지를 설치합니다. 순서대로 따라하세요.

### 2-1. VS Code 설치

1. 브라우저에서 [https://code.visualstudio.com](https://code.visualstudio.com) 접속
2. 파란색 **Download** 버튼 클릭 (Windows용 자동 감지됨)
3. 다운로드된 파일 실행 → **다음(Next)** 계속 클릭하여 설치 완료
4. 설치 중 **"PATH에 추가"** 옵션이 있으면 반드시 체크 ✅

### 2-2. Claude Code for VS Code 확장 설치

VS Code 안에서 바로 설치할 수 있습니다. 별도 프로그램을 다운로드할 필요가 없습니다.

1. VS Code를 실행합니다
2. 왼쪽 사이드바에서 **네모 블록 모양 아이콘** (확장, Extensions)을 클릭합니다  
   - 또는 키보드로 `Ctrl + Shift + X` 를 눌러도 됩니다
3. 상단 검색창에 **Claude Code** 를 입력합니다
4. 검색 결과에서 게시자(Publisher)가 **Anthropic** 인 항목을 찾습니다
5. 해당 항목의 **Install** 버튼을 클릭합니다

> ⚠️ **주의**: "Claude Code"로 검색하면 비공식 확장도 나올 수 있습니다.  
> 반드시 게시자(Publisher)가 **Anthropic** 인 것을 확인하세요.

6. 설치가 완료되면 왼쪽 사이드바에 **✱ (Spark 아이콘)** 이 새로 나타납니다
7. 이 아이콘이 보이면 설치 성공입니다!

**처음 실행 시 로그인:**

Spark 아이콘을 처음 클릭하면 브라우저가 열리면서 Anthropic 로그인 화면이 나타납니다.  
회사 계정으로 로그인하면 바로 사용할 수 있습니다. (처음 한 번만 하면 됩니다)

> 💡 **참고**: Claude Code는 유료입니다. Claude Max 구독 또는 API 키가 필요합니다.  
> 회사 계정 정보는 팀장에게 문의하세요.

## 3. 처음 실행하기

### 3-1. 작업 폴더 열기

1. VS Code 실행
2. 상단 메뉴에서 **파일(File)** → **폴더 열기(Open Folder)** 클릭
3. 기획 작업용 폴더를 선택합니다 (없으면 새로 만들기)
   - 예: `D:\미니게임즈\야구매니저-기획`
4. 폴더를 선택하면 VS Code 왼쪽에 해당 폴더의 파일 목록이 나타납니다

### 3-2. Claude Code 패널 열기

왼쪽 사이드바에서 **✱ (Spark 아이콘)** 을 클릭하면 Claude Code 채팅 패널이 열립니다.

> 💡 **Spark 아이콘을 찾는 3가지 위치:**
> - **왼쪽 사이드바**: 항상 보이는 ✱ 아이콘 (가장 쉬움)
> - **에디터 오른쪽 상단**: 파일을 열었을 때 나타나는 ✱ 아이콘
> - **하단 상태 표시줄**: 화면 맨 아래 **✱ Claude Code** 텍스트

### 3-3. 기본 사용법

Claude Code 채팅 패널 하단의 입력창에 **한국어로** 원하는 작업을 적으면 됩니다.  
카카오톡이나 메신저에 메시지 보내듯이 자연스럽게 작성하세요:

```
이 프로젝트의 폴더 구조를 보여줘
```

```
선수 성장 시스템 기획서 초안을 작성해줘
```

```
기존 투수 스탯 테이블을 참고해서 신규 선수 데이터를 만들어줘
```

**핵심**: 프로그래밍 명령어가 아닙니다. 그냥 하고 싶은 말을 하면 됩니다.

### 3-4. Claude가 파일을 수정할 때

Claude가 파일을 수정하면 **변경 전/후 비교 화면(diff)** 이 자동으로 나타납니다.

- 왼쪽(빨간색) = 기존 내용
- 오른쪽(초록색) = 변경된 내용

내용을 확인한 뒤:
- ✅ **Accept** 버튼 → 변경 사항을 적용합니다
- ❌ **Reject** 버튼 → 변경 사항을 취소합니다

> 💡 모든 수정은 이렇게 **내가 확인하고 승인**하는 과정을 거칩니다.  
> Claude가 함부로 파일을 바꾸지 않으니 안심하세요.

## 5. CLAUDE.md — AI에게 규칙 알려주기

**CLAUDE.md**는 프로젝트의 최상위 폴더에 놓는 특별한 파일입니다.  
Claude Code는 대화를 시작할 때 이 파일을 **가장 먼저 읽고**, 여기에 적힌 규칙을 따릅니다.

### CLAUDE.md 만드는 방법

**방법 1 — Claude에게 만들어달라고 하기 (가장 쉬움):**

Claude Code 채팅 패널에서:
```
프로젝트 루트에 CLAUDE.md 파일을 만들어줘. 아래 내용을 넣어줘:
(아래 예시 내용을 복사해서 붙여넣기)
```

**방법 2 — 직접 만들기:**

1. VS Code 왼쪽 파일 탐색기에서 빈 공간을 **마우스 오른쪽 클릭**
2. **새 파일(New File)** 선택
3. 파일명을 `CLAUDE.md` 로 입력하고 Enter
4. 열린 편집 화면에 아래 예시 내용을 붙여넣기
5. `Ctrl + S` 로 저장

### CLAUDE.md 작성 예시

아래 내용을 복사해서 자신의 프로젝트에 맞게 수정하세요:

```markdown
# 야구매니저 — 기획 프로젝트 규칙

## 프로젝트 개요
- 게임명: 야구매니저
- 장르: 모바일 야구 구단 경영 시뮬레이션
- 개발사: 주식회사 미니게임즈
- 핵심 키워드: 구단 운영, 선수 육성, 경기 시뮬레이션, 전략

## 응답 규칙
- 모든 응답과 문서는 **한국어**로 작성할 것
- 기획서는 마크다운(.md) 형식으로 작성할 것
- 파일명은 한글 가능, 공백 대신 하이픈(-) 사용
- 새 문서 작성 시 반드시 기존 문서를 먼저 참조할 것

## 게임 핵심 구조
- 핵심 루프: 선수 수집 → 육성 → 라인업 편성 → 경기 → 보상 → 강화
- 포지션: 투수(SP/RP/CP), 포수(C), 내야수(1B/2B/3B/SS), 외야수(LF/CF/RF), 지명타자(DH)
- 선수 등급: N → R → SR → SSR → UR
- 주요 스탯 (투수): 구위, 제구, 스태미나, 멘탈, 구종수
- 주요 스탯 (타자): 컨택, 파워, 선구안, 주루, 수비력, 체력

## 기획서 작성 규칙
- 기획서 상단에 항상 포함할 것:
  - 문서명
  - 작성일 / 수정일
  - 작성자
  - 버전
- 시스템 기획서에는 반드시 포함:
  - 개요 (이 시스템이 왜 필요한지)
  - 상세 설계
  - 수치 테이블 (해당 시)
  - 예외 처리
  - 연관 시스템
- 수치 공식을 작성할 때:
  - 공식은 명확한 수식으로 표기
  - 예시 계산을 최소 1개 포함
  - 관련 스탯의 범위(최솟값~최댓값)를 명시

## 선수 데이터 양식
선수 데이터 작성 시 아래 항목을 포함할 것:
- 선수명
- 포지션
- 등급 (N/R/SR/SSR/UR)
- 기본 스탯 테이블
- 스킬 목록 (패시브/액티브)
- 성장 곡선 (레벨별 스탯 증가치)
- 획득처 (가챠/이벤트/스토리 등)
- 선수 설명 (배경 스토리 1~2줄)

## 수치 설계 원칙
- 모든 확률은 소수점 둘째 자리까지 (예: 3.25%)
- 재화 수급량은 일간/주간/월간 기준으로 산출
- 밸런스 관련 수치 변경 시 반드시 밸런스-시트.md에 동기화할 것
- 과금 유저와 무과금 유저 간 격차 기준을 명시할 것

## 톤 & 문서 스타일
- 기획서 톤: 명확하고 간결한 서술체
- 불필요한 수식어 최소화
- 애매한 표현 금지 (예: "적당히", "꽤 많은" → 구체적 수치로)
```

### CLAUDE.md 수정 방법

이 파일은 언제든 수정할 수 있습니다.

- **직접 편집**: VS Code 왼쪽 탐색기에서 `CLAUDE.md` 를 클릭하면 편집 화면이 열립니다. 수정 후 `Ctrl + S` 로 저장하세요.
- **Claude에게 요청**: 채팅 패널에서 `CLAUDE.md에 시즌 구조 규칙을 추가해줘` 라고 입력하면 Claude가 대신 수정합니다.

> ⚠️ **중요**: CLAUDE.md를 수정한 뒤에는 Claude Code에서 **새 대화를 시작**해야  
> 변경사항이 반영됩니다.

## 6. 기획서 작성 실전

아래 예시들을 Claude Code 채팅 패널의 입력창에 그대로 복사해서 사용해보세요.

### 예시 1: 새로운 시스템 기획서 작성

```
02_선수 폴더에 선수 강화 시스템 기획서를 작성해줘.
기존 선수-성장.md와 등급-시스템.md를 참고해서 연관성 있게 만들어줘.
```

### Plan Mode — 수정 전에 계획만 먼저 확인하기

Claude Code 입력창 왼쪽에 **Plan** 토글이 있습니다.
이 모드를 켜면 Claude가 파일을 직접 수정하지 않고, **무엇을 어떻게 바꿀지 계획만 먼저 설명**합니다.

**사용 방법:**
1. 입력창 왼쪽의 **Plan** 버튼(또는 토글)을 클릭해서 Plan Mode 활성화
2. 평소처럼 작업을 요청합니다
3. Claude가 수정 계획을 텍스트로 보여줍니다 (파일은 건드리지 않음)
4. 계획이 마음에 들면 Plan Mode를 끄고 같은 요청을 다시 입력하세요

> 💡 **처음 사용하거나 큰 작업을 맡길 때 적극 활용하세요.**
> 결과물이 예상과 다르게 나오는 것을 사전에 방지할 수 있습니다.

## 8. 팁과 주의사항

### ✅ 이렇게 하면 좋습니다

1. **구체적으로 지시하기**
   - ❌ `기획서 써줘`
   - ✅ `03_시뮬레이션 폴더에 컨디션 시스템 기획서를 만들어줘. 피로도 누적 공식, 컨디션이 경기력에 미치는 영향, 회복 메커니즘 3개 섹션으로 나눠줘.`

2. **참고 문서를 명시하기**
   - ❌ `선수 데이터 만들어줘`
   - ✅ `스탯-체계.md와 등급-시스템.md를 참고해서 SSR 유격수 데이터를 만들어줘`

3. **작업 전 확인하기**
   - `이 폴더의 파일 목록을 보여줘` 로 현재 상태 확인
   - `타격-공식.md 내용을 요약해줘` 로 기존 내용 파악

4. **한 번에 하나씩**
   - 너무 많은 것을 한 번에 요청하면 품질이 떨어집니다
   - 큰 작업은 나눠서 지시하세요

5. **결과 확인 후 피드백**
   - `방금 만든 기획서에서 확률 테이블 부분을 더 세분화해줘`
   - `공식이 너무 복잡해. 변수를 줄여서 단순화해줘`

### ⚠️ 주의사항

1. **Claude Code가 파일을 직접 수정합니다**
   - 수정 시 반드시 diff 화면에서 **Accept/Reject** 를 확인하세요
   - 중요한 문서는 수정 전에 백업해두면 더 안전합니다
   - Git을 사용하면 더 안전합니다 (개발팀에 문의)

2. **CLAUDE.md가 없으면 맥락을 모릅니다**
   - 반드시 프로젝트 루트에 CLAUDE.md를 만들어두세요
   - 이 파일이 없으면 일반 AI 채팅과 크게 다르지 않습니다

3. **대화가 길어지면 앞부분을 잊을 수 있습니다**
   - 긴 작업은 중간중간 파일로 저장하면서 진행하세요
   - 새 대화를 시작해도 파일은 그대로 남아있으므로 안전합니다

4. **인터넷 실시간 정보는 조회하지 않습니다**
   - Claude Code는 기본적으로 **로컬 프로젝트 파일**을 기반으로 작업합니다
   - 최신 인터넷 정보를 실시간으로 조회하지는 않습니다
   - 실제 KBO/MLB 선수 데이터가 필요하면 별도로 조사하세요

5. **수치 공식은 반드시 검증하세요**
   - AI가 만든 공식이 수학적으로 맞더라도 게임 밸런스에 적합한지는 사람이 판단해야 합니다
   - 예시 계산을 요청해서 결과가 합리적인지 직접 확인하세요

6. **비용이 발생합니다**
   - 채팅 입력창에 `/cost` 를 입력하면 현재까지 사용한 비용을 확인할 수 있습니다
   - 불필요하게 긴 대화는 비용을 높입니다

## 9. 문제 해결 (FAQ)

### Q. Claude Code 확장이 검색되지 않습니다

**A.** VS Code 버전이 오래되었을 수 있습니다.  
상단 메뉴 **도움말(Help)** → **업데이트 확인(Check for Updates)** 으로 최신 버전인지 확인하세요.  
그래도 안 되면 [VS Code 마켓플레이스 웹사이트](https://marketplace.visualstudio.com/items?itemName=anthropic.claude-code)에서 직접 **Install** 버튼을 눌러 설치할 수 있습니다.

### Q. Spark 아이콘(✱)이 왼쪽에 보이지 않습니다

**A.** 아래 순서로 확인하세요:

1. 왼쪽 사이드바의 **네모 블록 아이콘** (또는 `Ctrl + Shift + X`)을 클릭하여 확장 패널을 엽니다
2. "Claude Code"가 **설치됨(Installed)** 목록에 있는지 확인합니다
3. 있다면 VS Code를 완전히 닫았다가 다시 실행해보세요
4. 그래도 안 보이면: `Ctrl + Shift + P` → `Developer: Reload Window` 입력 후 선택

### Q. Claude가 한국어가 아닌 영어로 답합니다

**A.** CLAUDE.md에 `모든 응답은 한국어로 작성할 것`이 포함되어 있는지 확인하세요.  
또는 채팅 패널에서 직접 지시하세요: `앞으로 모든 응답을 한국어로 해줘`

### Q. Claude가 기존 문서를 참고하지 않습니다

**A.** 파일을 직접 지정해주세요. 두 가지 방법이 있습니다:

**방법 1 — 텍스트로 경로 입력:**
```
02_선수/스탯-체계.md 파일을 읽고 참고해서 작업해줘
```

**방법 2 — @ 기호로 파일 선택 (더 쉬움):**  
채팅 입력창에 `@` 를 타이핑하면 파일 목록이 나타납니다.  
원하는 파일을 클릭하면 자동으로 참조됩니다.

### Q. 파일이 엉뚱한 곳에 생성되었습니다

**A.** 폴더 경로를 구체적으로 지정하세요:
```
03_시뮬레이션 폴더 안에 만들어줘
```
이미 잘못 생성된 파일은:
```
방금 만든 파일을 03_시뮬레이션 폴더로 옮겨줘
```

### Q. Claude가 만든 수치가 이상합니다

**A.** 전제 조건을 구체적으로 알려주세요:
```
타격 공식을 다시 계산해줘.
전제: SSR 타자 컨택 80 vs SR 투수 제구 70일 때
기대 타율이 0.280~0.320 사이가 나와야 해.
```

### Q. Claude가 만든 내용이 마음에 들지 않습니다

**A.** 구체적으로 피드백하세요:
```
방금 작성한 기획서에서:
1. 개요 부분이 너무 길어 → 3줄로 줄여줘
2. 확률 테이블에 비용 컬럼 추가해줘
3. 톤을 좀 더 간결하게 바꿔줘
```

### Q. VS Code 왼쪽에 파일이 안 보입니다

**A.** 올바른 폴더가 열려있는지 확인하세요:  
상단 메뉴 **파일(File)** → **폴더 열기(Open Folder)** → 기획 프로젝트 폴더 선택

단축키를 외우지 않아도 **상단 메뉴나 아이콘 클릭**으로 같은 기능을 사용할 수 있습니다.

| 단축키 | 기능 | 마우스로 하는 법 |
|---|---|---|

| `Ctrl + P` | 파일명으로 빠르게 찾기 | — |
| `Ctrl + Shift + F` | 전체 파일에서 텍스트 검색 | 왼쪽 사이드바 돋보기 아이콘(🔍) 클릭 |
| `Ctrl + S` | 현재 파일 저장 | 상단 메뉴 **파일(File)** → **저장(Save)** |
| `Ctrl + Z` | 실행 취소 | 상단 메뉴 **편집(Edit)** → **실행 취소(Undo)** |
| `Ctrl + Tab` | 열린 파일 간 전환 | 상단 탭 직접 클릭 |

## 부록 B: 터미널이 필요한 경우

> **이 섹션은 건너뛰어도 됩니다.** 일상적인 기획 작업에는 터미널이 필요하지 않습니다.
> 개발팀 또는 기술지원팀의 안내가 있을 때만 참고하세요.

> 🖥️ **터미널**이란?  
> VS Code 안에 내장된 **명령어 입력 화면**입니다.  
> 검은색(또는 어두운 색) 배경에 텍스트를 입력하는 창으로,  
> 별도 프로그램을 열 필요 없이 VS Code 안에서 바로 사용할 수 있습니다.

일상적인 기획 작업은 **Claude Code 채팅 패널만으로 충분**합니다.  
하지만 아주 드물게 터미널이 필요한 경우가 있습니다:

- 개발팀이 특정 명령어 실행을 요청했을 때
- 문제 해결을 위해 기술지원팀이 안내하는 명령어를 실행할 때

### VS Code에서 터미널 여는 법

**방법 1 — 메뉴 클릭 (추천):**  
상단 메뉴에서 **터미널(Terminal)** → **새 터미널(New Terminal)** 클릭

**방법 2 — 단축키:**  
키보드에서 `` Ctrl + ` `` 누르기  
(`` ` `` 키 위치: 키보드 왼쪽 상단, **ESC 키 바로 아래**에 있는 작은 키입니다)

터미널이 열리면 VS Code **하단**에 입력창이 나타납니다.  
여기에 명령어를 입력하고 **Enter** 를 누르면 실행됩니다.
